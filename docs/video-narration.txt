PROOFLINE — 90-SECOND DEMO SCRIPT

Before recording
- Run: npm run app -- --demo
- Open: http://127.0.0.1:4173
- Use one browser window at 100% zoom; silence notifications.
- Start on Overview with the DEMO SESSION badge visible.
- Keep the Codex feedback receipt and `npm test` result ready in separate tabs.

0:00–0:10 — The problem
Screen: Overview. Keep the demo badge and empty review trail visible.
Say: “Reviewer feedback names a problem, not the paper that proves it.”
Say: “Proofline turns that comment into an evidence trail the researcher can inspect.”

0:10–0:18 — The connected workspace
Screen: Open Connections. Hold on the VS Code, Obsidian, and Overleaf cards.
Say: “Proofline connects the work around research: VS Code, Obsidian, and Overleaf.”
Say: “Each is optional, project-scoped, and stays under the researcher’s control.”

0:18–0:23 — Set up the wow moment
Screen: Click “Show completed proof.” Pause while the Evidence notes view loads.
Say: “For judges, this deterministic replay opens the complete, approval-gated path in one click.”

0:23–0:38 — Claim Traceback (primary wow moment)
Screen: Hold on Claim Traceback. Slowly reveal its four steps.
Pause: 2 seconds before speaking.
Say: “This is the proof: one draft statement, its selected paper, approved task, and original feedback.”
Say: “Every link is drawn from canonical local state—not a generated story.”

0:38–0:49 — Grounded note
Screen: Show the grounded note preview and the stable source ID.
Say: “The note only uses papers the researcher selected as evidence.”
Say: “In a real vault, Obsidian receives this note only after a separate approval.”

0:49–1:01 — Boundary proof
Screen: Click “Test citation boundary.” Hold on the green rejection confirmation.
Pause: 2 seconds after the confirmation appears.
Say: “Now I inject an unselected citation. Proofline refuses it before preview.”
Say: “That boundary is enforced, demonstrated, and covered by tests.”

1:01–1:12 — Review trail
Screen: Return to Overview and open the Revision Response Matrix.
Say: “The response matrix gives a supervisor the same trail: comment, task, evidence, and note status.”

1:12–1:22 — Build proof
Screen: Show the Codex feedback receipt, then the passing `npm test` terminal result.
Say: “Codex with GPT-5.6 built the submitted project. Runtime model choice stays separate and visible.”

1:22–1:30 — Close
Screen: Return to Claim Traceback.
Say: “Chat tools generate text. Proofline generates proof: every citation comes from evidence you approved.”

Recording checklist
- The “DEMO SESSION · ISOLATED DATA” label is visible at least once.
- Claim Traceback appears before the halfway point.
- The citation-rejection confirmation is readable and held for two seconds.
- Do not claim that the demo writes to Obsidian; it deliberately cannot.
- Say that Overleaf is URL-only: Proofline opens the researcher’s project but never signs in or syncs it.
- Export or show the response matrix only after explaining it is read-only.
