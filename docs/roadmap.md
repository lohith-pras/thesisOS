# Adapter roadmap

## Implemented

- Zotero Desktop read-only discovery, selection, and search
- Codex CLI decomposition and grounded drafting
- Optional OpenAI decomposition and drafting adapters
- Obsidian Markdown preview and explicit local write
- Local semantic retrieval with metadata fallback
- Provenance-aware Paper Card and hierarchical Paper Map API, with no inferred paper claims
- Read-only Obsidian vault audit for broken links and managed-note provenance gaps

## Planned

- Paper Map UI and researcher review form for verified question, method, data, findings, limitations, and thesis relevance
- One note per selected paper plus a linked evidence index
- PDF full-text extraction for missing abstracts, with page/section provenance
- User-maintained relevance judgments
- Approval-gated vault maintenance actions for accepted audit proposals
- Evidence-backed argument and chapter-outline alternatives for the LaTeX agent
- Reviewable LaTeX/Overleaf Git patch adapter
- Reviewable thesis-state diffs
- Local Git/Overleaf patch adapters
- Experiment planning and result inspection through VS Code
- arXiv discovery before Zotero import

The current submission intentionally presents the implemented vertical slice rather than implying that planned adapters are available.
