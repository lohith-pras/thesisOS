ThesisOS onboarding: show the destination, then personalize the path

A seven-step form would feel like configuration. This version behaves like a guided conversation and produces a useful Overview immediately.

The user-visible sequence
1 · SHOW THE OUTCOME

“Turn thesis intent and supervisor feedback into evidence-backed next actions.”

30-second product preview
2 · NAME THE THESIS

“What should we call your thesis?”

Only required answer
3 · UNDERSTAND THE WORK

Add project PDF, choose stage, state current focus.

All skippable
4 · BUILD THE RESEARCH BASE

Connect Zotero when papers become relevant.

Benefit shown before permission
5 · CHOOSE OUTPUTS

Optional manuscript folder and Obsidian vault.

Explain paths in plain language
6 · PERSONALIZED PREVIEW

Show their real status, connections, next step, and profile gaps.

Answers visibly pay off
7 · ENTER THE GUIDED LIFECYCLE

Your thesis workspace is ready enough to begin.

Incomplete integrations remain visible as recommended next steps—not errors. The first action is adding supervisor feedback with the approved profile context available.

What changes from the earlier wizard

  • Outcome preview comes before questions
  • Questions are grouped by user goal, not by adapter
  • Integrations are requested only after their benefit is clear
  • The final screen previews the actual personalized Overview

Deliberate constraints

  • No fake completion percentage
  • No blocking “complete your setup” state
  • No claim that Overleaf is connected when only a local folder exists
  • No requirement beyond thesis name
